## Tools

You must have at least 3 tools. The three required tools are listed — add any additional tools below them.

### Tool 1: search_listings

**What it does:**
Searches the static mock listings dataset for items matching the user's criteria (description keywords, max price, and size) and returns the matching listings.

**Input parameters:**
- `description` (str): Keyword description of the item the user is looking for.
- `size` (str | None): The user's requested size (e.g., "M", "XXS"). Optional.
- `max_price` (float | None): The maximum price the user is willing to pay. Optional.

**What it returns:**
A list of dictionaries `list[dict]`, where each dictionary represents a matched listing.

**What happens if it fails or returns nothing:**
If no listings match, it returns an empty list `[]`. The planning loop will catch this and initiate **Retry Logic with Fallback** (e.g., dropping the size constraint and trying again). If the retry also fails, it early-returns an error to the user.

---

### Tool 2: suggest_outfit

**What it does:**
Uses the Groq LLM to suggest one or more complete outfit combinations by pairing the newly found item with existing pieces from the user's wardrobe, influenced by current trends.

**Input parameters:**
- `new_item` (dict): The selected listing dictionary returned from `search_listings`.
- `wardrobe` (dict): The user's current wardrobe dictionary.
- `trends` (str | None): Optional context about current trends from `check_trends`.

**What it returns:**
A string `str` containing the LLM-generated outfit suggestion.

**What happens if it fails or returns nothing:**
If the wardrobe is empty, the tool will rely on the LLM to suggest general pairing advice instead of relying on specific wardrobe pieces.

---

### Tool 3: create_fit_card

**What it does:**
Uses the Groq LLM to generate a short, shareable social-media-style caption ("fit card") describing the complete outfit.

**Input parameters:**
- `outfit` (str): The outfit suggestion generated by the `suggest_outfit` tool.
- `new_item` (dict): The listing dictionary of the newly found item.

**What it returns:**
A string `str` containing the generated caption.

**What happens if it fails or returns nothing:**
If `outfit` is empty, generates a generic caption about the item alone or returns a fallback message.

## Planning Loop

**How does your agent decide which tool to call next?**
The agent uses a sequential planning loop with fallback behavior. 
1. **Style Profile Memory (Stretch Feature):** The agent loads any previously saved style preferences from a persistent local JSON file (`data/style_profile.json`).
2. **Search:** The agent parses the query and calls `search_listings()`. 
3. **Retry Logic (Stretch Feature):** If `search_results` is empty, it retries the search with loosened constraints (e.g., drops `size` or `max_price`), storing a note in the session that constraints were relaxed. If it is still empty, it halts execution and returns an error.
4. **Analysis:** It calls `compare_price()` and `check_trends()` for the selected item (`results[0]`).
5. **Suggestion:** It calls `suggest_outfit()`, passing in the item, wardrobe, and trends.
6. **Caption:** It calls `create_fit_card()`, storing the final caption.
7. **Profile Update:** Updates and saves the `style_profile.json` with the new preferences.

---

## State Management

**How does information from one tool get passed to the next?**
A single `session` dictionary tracks the state. Additionally, `data/style_profile.json` will be used for cross-session persistent state. `session["search_results"]` holds the search items, `session["fallback_used"]` tracks if constraints were loosened, `session["price_comparison"]` and `session["trends"]` store analysis data. All of this flows into `suggest_outfit` and `create_fit_card`.

## Error Handling

For each tool, describe the specific failure mode you're handling and what the agent does in response.

| Tool | Failure mode | Agent response |
|------|-------------|----------------|
| search_listings | No results match the query | Agent activates retry logic by dropping constraints (e.g. size). If still no results, halts loop and returns an informative error explaining what failed. |
| suggest_outfit | Wardrobe is empty | The prompt instructs the LLM to provide general styling advice for the item instead of relying on specific wardrobe pieces. |
| create_fit_card | Outfit input is missing or incomplete | Generates a generic caption about the item alone or returns a fallback message. |
| compare_price | No comparable items | Skips the comparison and returns "Unable to compare price." |
| check_trends | No trend data | Falls back to generic outfit suggestions without trend influence. |

## A Complete Interaction (Step by Step)

Write out what a full user interaction looks like from start to finish — tool call by tool call. Use a specific example query.

**Example user query:** "I'm looking for a vintage graphic tee under $10. I mostly wear baggy jeans and chunky sneakers. What's out there and how would I style it?"

**Step 1:** The agent loads the user's style profile memory. It parses the query (`description="vintage graphic tee"`, `max_price=10.0`). It calls `search_listings`.

**Step 2:** `search_listings` returns an empty list because $10 is too low. The **Retry Logic** activates, removing `max_price`, and re-calls `search_listings`. This succeeds, finding a "Faded Band Tee — $22". The agent notes the fallback in the session.

**Step 3:** The agent calls `compare_price()`, which reports "$22 is a fair price (average for vintage tees is $25)".

**Step 4:** The agent calls `check_trends()`, which reports "Baggy 90s aesthetic is currently trending."

**Step 5:** The agent calls `suggest_outfit()`. The LLM incorporates the new tee, the wardrobe (baggy jeans), the trends, and notes the relaxed budget constraint.

**Step 6:** The agent calls `create_fit_card()` using the outfit suggestion.

**Step 7:** The agent saves the parsed preferences (baggy jeans, chunky sneakers) to the persistent style profile and displays the final output to the user.

**Final output to user:**
The user sees the top listing details (Faded Band Tee, $22) with a note that the price constraint was relaxed, a price fairness estimate, the generated outfit idea involving their baggy jeans and sneakers (influenced by 90s trends), and a shareable fit card caption.
